What the run does

  1. Pulls machine-readable PayBridge snapshots from /api/ops/transactions/export.
  2. Stores local snapshot rows for the run.
  3. Matches settlement rows by provider payment ID, provider transaction ID, then order ID.
  4. Creates SETTLEMENT_ONLY, PAYBRIDGE_ONLY, AMOUNT_MISMATCH, and DUPLICATE_SETTLEMENT_ROW cases.
